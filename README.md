# 券商板块行情启动信号回测系统

基于中证证券公司指数（399975.SZ）历史行情的多轮技术指标回测与优选赔率筛选工具。帮你回答三个问题：**行情是不是要来了？来了之后买哪只？什么时候卖？**

## 功能

- **行情周期自动检测**：从 2009-2025 年指数数据中识别出 9 轮牛市周期（`end_date` 采用区间实际最高收盘价）
- **14 种买入指标回测**：MACD/RSI/CCI/KDJ/OBV/布林带/ADX 等，多参数网格搜索 + 日线/周线/共振三种频率
- **14 种卖出指标回测**：MACD顶背离/RSI顶背离/CCI超买回落/布林带上轨触顶 等，捕获率评估（≥0.90 为命中）
- **买入+卖出联合回测**：完整交易回合模拟，计算每轮行情的实际收益率与捕获率
- **49 只成分股赔率排名**：每轮周期个股弹性 Z 分评估，排出历史上谁涨得多
- **个股买入/卖出回测**：Top 指标逐股评估，按有效周期统计命中率与信号有效率
- **实时信号检测**：Top 3 买入/卖出指标当天信号扫描，趋势上下文过滤
- **信号 × 赔率交叉筛选**：星级信号评级 + 一键筛选，直接找出"高赔率 + 已触发"的标的
- **领涨/滞涨节奏分析**：每轮行情启动后前 N 个交易日涨幅排序，检验"谁先涨"是否跨周期稳定
- **Streamlit 交互界面**：9 个 Tab，K 线图 + 指标叠加 + 热力图 + 排名表

## 快速开始

```bash
# 1. 安装依赖
pip install -r requirements.txt

# 2. 设置 tushare token（需要 2000+ 积分）
echo "你的token" > .tushare_token

# 3. 首次运行：拉数据 + 检测周期 + 回测 + 算赔率
python detect_cycles.py
python screener.py --init

# 4. 启动 Web 界面
streamlit run app.py
```

## 项目结构

| 文件 | 用途 |
|------|------|
| `app.py` | Streamlit Web 主界面（9 个 Tab） |
| `indicators.py` | 14 种买入 + 14 种卖出技术指标计算 + 上下文过滤 |
| `backtest.py` | 买入/卖出回测引擎 + 联合回测 + 个股回测 |
| `screener.py` | 个股赔率 Z 分筛选 |
| `data_fetcher.py` | tushare 主 + AKShare 备双源数据拉取 + 节假日 API + 缓存 |
| `detect_cycles.py` | 行情周期检测（使用实际最高收盘价） |
| `lead_lag.py` | 领涨/滞涨节奏分析（跨周期涨幅排序稳定性） |
| `test_indicators.py` | 买入指标单元测试（29 个用例） |
| `test_sell_indicators.py` | 卖出指标单元测试（48 个用例） |
| `data/` | 本地数据缓存（已 gitignore） |
| `output/` | 回测结果 + 赔率排名（已 gitignore） |
| `docs/` | 实施计划文档 |

## 使用流程

1. **侧边栏加载数据** → 拉取指数 + 49 只成分股日线
2. **点击「重新回测（买入+卖出）」** → 跑全部指标 × 参数组合的批量回测
3. **Tab 2 买入指标回测** → 看买入信号命中率排名
4. **Tab 7 卖出指标回测** → 看卖出信号捕获率排名
5. **Tab 5 行情跟踪** → 看当前买入/卖出信号状态
6. **Tab 6 优选赔率** → 勾选「仅显示有信号」，排名前两名即候选
7. **Tab 8 买卖联合评估** → 逐轮完整交易模拟结果
8. **Tab 7 领涨/滞涨分析** → 看当前轮次谁先涨，对照历史规律判断节奏

## 依赖

- Python 3.10+
- AKShare（备数据源，A股行情）
- Tushare Pro（主数据源，需 2000+ 积分）
- requests（节假日 API 调用）
- TA-Lib（需要系统级安装，macOS 用 `brew install ta-lib`）
