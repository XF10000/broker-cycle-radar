# 券商个股赔率筛选 — 设计文档

## 目标

在已有买点信号检测系统基础上，新增**个股赔率筛选**功能。回答核心问题：

> 历史上券商板块牛市时，哪几只个股涨得最凶？

筛选器不作为买卖决策，而是提供数据驱动的赔率对比，辅助用户从全量券商成分股中识别高赔率标的。

---

## 数据获取

### 个股范围

通过 tushare 获取中证证券公司指数（399975.SZ）的**实时成分股列表**。

- 接口：`pro.index_weight()` 或 `pro.index_member()`（需确认2000积分下具体可用接口）
- 优势：获取的是当前市场存续、流动性正常的个股，符合实际投资需求
- 不采用行业分类方式，因为会混入已退市、数据不可得的标的，增加无效数据量

### 个股日线数据

对成分股列表中的每只个股，通过已有 `data_fetcher.py` 的缓存逻辑拉取日线（复用现有 `pro.daily()` + CSV本地缓存机制）。

### 周期窗口

使用已有 `output/cycles.csv` 中检测的券商板块牛熊周期（2010年后共9轮），以指数周期起止点作为统一时间窗口，保证截面可比。

### 板块指数基准

使用 `data/index_daily.csv`（399975.SZ 日线）作为板块涨跌幅基准。

## 核心指标：周期内 Z 评分

### 为什么用 Z 评分而不是绝对涨幅或超额收益

1. **绝对涨幅在不同周期不可比**：一轮 +25% 行情 vs 一轮 +173% 行情，+5% 超额在后者微不足道，在前者却是大胜。
2. **超额收益绝对值不可比**：同上原因，不同周期幅度差异扭曲了超额收益的含义。
3. **Z 评分天然消除周期差异**：每轮内部做标准化，使跨周期汇总变得有效。

### 计算步骤

对每一轮牛熊周期（以 `output/cycles.csv` 定义的时间窗口为界）：

1. **确定参与个股集合**：该周期内存在日线数据的所有成分股
2. **计算每只个股的周期内涨幅**：
   - 定义：`（周期内最高收盘价 / 周期起点收盘价）- 1`
   - 取最高收盘价而非周期终点收盘价，因为核心问题是"它最多能涨多少"，不要求用户精准抄底逃顶
3. **对涨幅集合做标准化**：
   - μ = 该周期内所有个股涨幅的均值
   - σ = 该周期内所有个股涨幅的标准差
   - Z_i = (涨幅_i - μ) / σ
4. **含义**：Z = +1.5 意味着该股在该轮周期里的表现比同辈平均高了 1.5 个标准差，是显著的大胜

### 跨周期汇总

对每只个股，取所有参与周期的 Z 评分，计算：

| 指标 | 计算方式 | 含义 |
|---|---|---|
| **中位数 Z** | 所有参与周期 Z 评分的中位数 | 该股的"常态"超额水平。主排序依据。 |
| **Z 最高值** | 所有参与周期 Z 评分的最大值 | 该股的"爆发力"上限。最佳表现。 |
| **Z 正值率** | Z > 0 的周期数 / 总参与周期数 | 该股在多少比例的周期里跑赢了同行中位数。反映稳定性。 |
| **中位涨幅** | 所有参与周期原始涨幅的中位数 | 直觉参考值，不用于排序 |
| **最大涨幅** | 所有参与周期原始涨幅的最大值 | 直觉参考值，不用于排序 |
| **跑赢指数概率** | 个股涨幅 > 板块指数涨幅 的周期数 / 总参与周期数 | 简洁直观的对比指标 |

### 为什么用中位数而不是均值

- 9轮周期中存在极端值（2014-2015年 +173%），均值会被极端值过度影响
- 中位数对异常值稳健，更能反映该股的"常态"表现

## 置信度标签

### 问题

不同个股上市时间不同，参与周期数从1轮到9轮不等。参与轮数少的个股，统计结论置信度天然更低。

### 规则（数据驱动，非预设）

不预设阈值（如"≥5轮=可信"），而是在首次运行 `screener.py --init` 获取全量成分股后，基于实际数据分布确定：

**确定过程：**
1. 拉取全量成分股列表
2. 对每只个股统计其在历史周期中的参与轮数
3. 对全量个股的参与轮数取中位数，设为 M
4. 分级规则自动确定：
   - **可信**：参与轮数 ≥ M（样本量不低于行业中位数水平）
   - **参考**：参与轮数 ≥ 2 且 < M（样本偏少但仍有统计意义）
   - **数据有限**：参与轮数 < 2（仅1轮或0轮，无统计意义，仅供查看）

M 值记录于 `stock_odds.csv` 头部元信息行。日常 `--refresh` 不重新计算 M，仅在 `--refresh-index`（成分股变化）或 `--init`（首次构建）时重新确定。详见「数据刷新机制」章节。

## 共同窗口对比

### 问题

Z 评分已解决跨周期可比性，但用户希望有一个"公平窗口"——只取大家都参与的周期，在这个子集里直接对比原始涨幅，作为直觉验证。

### 规则（数据驱动）

取最近 N 轮连续周期，使得"参与率"（即成分股中有 ≥ 50% 的个股参与了该轮）最大，同时满足：
- N ≥ 2（至少两轮才有对比意义）
- 尽可能覆盖每只个股

实际 N 的确定方式：
1. 统计每轮周期的个股参与数
2. 从最近一轮往前累加，直到参与率跌破 50% 的那轮为止
3. N = 满足条件的连续轮数

## 信号状态联动

已有指标检测系统（14个技术指标 × 62条规则）为当前个股提供买点信号。排名表中的"信号"列引用此系统。

### 职责分离

`screener.py` 只做赔率计算，不嵌入信号检测逻辑。信号标注在 `app.py` 的 Tab 5 加载时完成：

1. `app.py` 加载 `stock_odds.csv`
2. 对表中每只个股，跑已有信号检测流程（与 Tab 4 "行情跟踪"相同逻辑）
3. 标注：▲ = 当前有买点触发，○ = 无信号，▼ = 无买点/信号已过期
4. 首次加载显示进度条，结果缓存到 `output/signal_cache.csv`
5. 日线数据更新时自动失效缓存

### 性能

- 全量成分股预计 20-50 只
- 每只需跑 14 个指标 × ~62 条规则
- 首次加载估计耗时：25-60 秒（含进度条）
- 缓存命中后：秒级

## 数据刷新机制

| 数据层 | 来源 | 刷新频率 | 命令 |
|---|---|---|---|
| 成分股列表 | tushare | 每月一次 | `screener.py --refresh-index` |
| 个股日线 | tushare | 每日收盘后 | `screener.py --refresh`（增量拉取） |
| Z评分 | 本地重算 | 日线更新后自动 | 随 `--refresh` 自动执行 |
| 信号缓存 | 本地重算 | 日线更新后自动 | app.py 检测日线更新时自动失效旧缓存 |

### 阈值（M/N）的重算规则

M（置信度中位数）和 N（共同窗口轮数）在 `--init` 时首次确定，写入 `stock_odds.csv` 头部元信息。后续行为：

- `--refresh`：读取现有阈值，**不重新计算**。避免日常刷新导致阈值波动，使用户看到的标签前后不一致。
- `--refresh-index`：成分股变化时**重新计算**阈值，因为个股集合变了，分布也需要更新。
- `--init`：如果目标文件已存在，**覆盖**写入全新阈值。

> 设计意图：阈值反映的是"当前成分股集合下的统计分布"，不应因每日数据刷新而漂移。仅在成分股集合变化时才需要重新评估。

### 命令

```bash
python screener.py --init           # 首次：拉取成分股 + 全量日线 + 计算Z评分及阈值
python screener.py --refresh        # 日常：增量日线 + 重算Z评分（阈值不变）
python screener.py --calc-only      # 仅重算Z（日线已最新）
python screener.py --refresh-index  # 重新拉取成分股 + 重新计算阈值
```

## 输出文件

### `output/stock_odds.csv`

每行一只个股，列：

| 列名 | 类型 | 说明 |
|---|---|---|
| `ts_code` | str | 个股代码 |
| `name` | str | 证券名称 |
| `median_z` | float | 中位数Z评分 |
| `max_z` | float | Z最高值 |
| `positive_z_rate` | float | Z正值率（0-1） |
| `median_return` | float | 中位涨幅（小数形式） |
| `max_return` | float | 最大涨幅（小数形式） |
| `beat_index_rate` | float | 跑赢指数概率（0-1） |
| `cycle_count` | int | 参与周期数 |
| `confidence` | str | 置信度标签（"可信"/"参考"/"有限"） |
| `float_mv` | float | 流通市值（万元，可能为空） |

文件头部包含元信息行（以 `#` 开头）：
- 置信度阈值 M 的实际值
- 共同窗口 N 的实际值
- 数据更新时间

## UI设计

### 位置：app.py 新增第5个 Tab「优选赔率」

### 顶部状态栏

```
数据更新至：YYYY-MM-DD | 覆盖券商：X只 | Z评分：Y只达标 | [刷新数据]
```

### 主视图：赔率排名表

默认按中位数Z降序排列，表头支持点击切换排序。

| 列 | 内容 |
|---|---|
| # | 序号 |
| 名称 | 证券名称 |
| 中位数Z | 主排序列 |
| Z最高值 | 爆发力 |
| Z正值率 | 稳定性 |
| 中位涨幅 | 原始% |
| 最大涨幅 | 原始% |
| 跑赢概率 | 直观% |
| 流通市值 | 参考 |
| 置信度 | 标签 |
| 轮数 | 参与数 |
| 信号 | ▲/○/▼ |
| 关注 | 切换按钮 |

- **关注按钮**：纯本地标注，存储于 `streamlit.session_state`，不影响计算。点击切换"关注"/"已关注"。
- **信号颜色**：▲买点=绿色、○无信号=灰色、▼无买点=浅红

### 辅助视图（Table内折叠区域）

- **Z分布箱线图**：每只个股的Z评分分布（Plotly box plot），显示中位数、四分位距、异常值
- **共同窗口涨幅对比**：柱状图，最近N轮内所有个股的原始涨幅直接对比

### 展开明细（点击行展开）

展开某一行的逐轮周期明细表：周期编号、起止日期、板块涨幅、个股涨幅、Z评分
