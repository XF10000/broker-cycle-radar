# 信号评级与交叉筛选 — 设计文档

## 1. 背景

当前系统两个问题：

1. **信号列二元 ▲/○ 有误导性**："优选赔率"Tab 和"行情跟踪"Tab 中，只要 Top 3 指标（CCI、MACD柱线、OBV）中任意一个在近 90 天内触发，就显示 ▲ 买点。三个指标的有效率不同（7/9、7/9、5/9），但 UI 对此不做区分。

2. **缺少"赔率 × 信号"交叉视图**：赔率排名和信号状态各自独立展示，用户需要手动在两页之间对照，才能找到"高 Z 分 + 当前信号触发"的股票。

3. **图表缺少图例**：行情跟踪、指标回测排行榜、个股回测对比三个 Tab 的 K 线图中，指标线（CCI、DIF/DEA、OBV 等）没有图例标注，新用户无法识别各线含义。

## 2. 改动范围

仅修改 `app.py` 中展示层函数，不涉及 `screener.py`、`backtest.py`、`indicators.py` 等核心逻辑。

## 3. 改动详情

### 3.1 信号星级评级

`compute_odds_signals()` 已有 `sig_count` 字段（0-3），目前只做了 `>=1 → 'buy'` 的二元映射。修改调用处的展示函数，按触发指标数量展示星级：

| sig_count | 展示 | 含义 |
|-----------|------|------|
| 3 | ★★★ | 三信号共振（CCI+OBV+MACD 全触发） |
| 2 | ★★☆ | 双信号确认 |
| 1 | ★☆☆ | 单一信号 |
| 0 | —— | 无信号 |

涉及位置：

- **Tab 5 `render_odds_tab()`**：`fmt_signal()` 函数改为星标，替换当前的 ▲/○；表头说明文字同步更新
- **Tab 4 `_render_signal_summary()`**：从"每股票每指标一行"改为"每股票一行"，增加信号星级、触发指标列表、最近信号日列，按信号数量优先排序

### 3.2 优选赔率页信号筛选

在 Tab 5 赔率排名表上方新增 checkbox "仅显示当前有信号"。勾选时只显示有 ★ 的股票，仍按中位数 Z 降序排列，并显示筛选结果数量。

### 3.3 图表图例

三个函数均从 `showlegend=False` 改为 `showlegend=True`，蜡烛图单独设 `showlegend=False`（K 线本身无需图例）：

- **`_render_live_chart()`**：行情跟踪页的实时 K 线图，图例横向置于底部
- **`_render_cycle_detail()`**：指标回测排行榜的各周期详情图，图例横向置于底部（字体缩小适应多图布局）
- **`_render_stock_cycle_detail()`**：个股回测对比的各股票周期详情图，同上

### 3.4 图表说明文字

- 优选赔率页"中位数Z评分"图表下方增加说明：Z>0 弹性好，Z<0 弹性差
- 优选赔率页"中位涨幅%"图表下方增加说明：解释共同窗口含义

## 4. 技术说明

- `compute_odds_signals()` 返回的 DataFrame 已有 `sig_count` 列，无需修改此函数
- `st.session_state.odds_signal_cache` 中已缓存 `sig_count`，无需重复计算
- 星级展示用 Unicode 字符 `★` `☆` `——`，无需额外图标库
- 图例位置统一使用 `legend=dict(orientation='h', ...)` 横向布局

## 5. 验证

- `streamlit run app.py` 后可直观检查
- 不需自动化测试（纯 UI 展示变化）
