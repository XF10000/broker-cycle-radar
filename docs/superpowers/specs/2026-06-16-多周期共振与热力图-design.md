# 多周期共振 + 热力图 — 设计文档

## 1. 目标

在现有日线/周线独立回测基础上，增加两项能力：

1. **多周期共振回测**：周线指标先出信号 → 打开观察窗口 → 日线指标确认，才算有效。模拟实战中"大周期定方向、小周期找买点"。
2. **热力图**：一张图展示所有指标 × 所有行情的命中情况，颜色区分命中/太晚/错过，一眼看出指标稳定性。

## 2. 共振回测规则

### 2.1 信号组合

从排行榜前列取候选指标：
- 周线 TOP 3（默认）：按综合得分取日线排行前列中周期为周线的指标
- 日线 TOP 3（默认）：按综合得分取日线排行前列中周期为日线的指标

对每种 (周线A, 日线B) 组合，生成一条「共振规则」，作为新的回测条目。

### 2.2 共振判定

```
周线 A 在日期 S_w 触发
  → 打开观察窗口 [S_w, S_w + 30 交易日]
  → 窗口内日线 B 在日期 S_d 触发
  → 共振信号确认，信号日为 S_d（日线触发日）
```

对于每轮行情 (启动日 T)：
- 如果 S_d 落在 [T-45, T] → 命中，记录提前天数 = T - S_d
- 如果 S_d > T+10 → 太晚
- 否则 → 错过

### 2.3 假信号与得分

共振的假信号 = 共振信号落在所有行情窗口之外的次数。得分公式和现有回测一致：`命中率×0.4 + 有效率×0.3 - 时间偏差×0.3`。

### 2.4 输出

共振结果追加到 `output/backtest_results.csv`，新增行，规则名格式：`周线:MACD柱线缩短 + 日线:CCI反转`。

## 3. 热力图

### 3.1 布局

位于排行榜标签页下方，标题「指标 × 行情 热力图」。

### 3.2 横轴

8 轮行情，每列标注起止日期和涨幅。

### 3.3 纵轴

所有日线 + 周线指标规则（排除资金流向类），按综合得分降序排列。

### 3.4 格子颜色

```
行情前45天窗口内 → 有信号 → 提前 1-15 天 → 🟢 深绿
                          → 提前 16-45 天 → 🟢 浅绿
                 → 无信号 → 行情启动后10天内有 → 🟡 黄色（太晚）
                          → 完全没有 → 🔴 红色（错过）
```

### 3.5 交互

- 鼠标悬停：显示指标名、行情名、命中状态、提前天数
- 点击格子：弹出该指标在该行情的 K 线细节图（复用现有 `_render_cycle_detail`）

### 3.6 技术实现

用 Plotly `go.Heatmap`，自定义 `colorscale`，`hovertemplate` 显示详情。`clickData` 回调触发 `_render_cycle_detail`。

## 4. 现有个股验证页调整

侧边栏新增按钮「仅看共振信号」，开启后个股验证页只展示共振规则的结果（周线+日线都触发的组合）。

## 5. 文件改动

- `backtest.py`：新增 `run_resonance_backtest()` 函数
- `app.py`：标签页 2 新增热力图区域 + 共振结果行；侧边栏加共振开关

现有功能不受影响。

## 6. 约束

- 共振组合最多 3×3=9 种，避免组合爆炸
- 热力图刷新实时计算，不缓存中间文件
- 共振不替代独立回测，作为补充维度存在
