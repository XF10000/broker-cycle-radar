# 券商板块技术指标回测系统 — 设计文档

## 目标

从 A 股证券板块过去 10 年的 4~5 轮行情中，系统性地回测技术指标的预警效果，找到在行情启动前发出的最有效信号，辅助未来的买入决策。

用户计划：1/3 资金买 ETF，2/3 资金分两份买两只候选个股（中信建投、中信证券、中泰证券、中银证券、财达证券）。

## 技术栈

- Python 3 + Streamlit（交互界面）
- tushare API（2000 积分，数据源）
- Plotly（K 线图与指标可视化）
- pandas / numpy / scipy（数据处理与信号检测）

## 项目结构

```
券商ETF3/
├── data/                  # 缓存 CSV（交易日线 + 周线）
├── output/                # 回测结果缓存
│   ├── cycles.csv
│   └── backtest_results.csv
├── indicators.py          # 技术指标计算函数库
└── app.py                 # Streamlit 入口（数据拉取、周期识别、回测、界面）
```

单命令启动：`streamlit run app.py`

数据拉取首次执行后缓存到 `data/`，后续启动直接读缓存。界面提供「刷新数据」按钮手动强制更新。

## 数据源

通过 tushare 拉取：
- 证券板块指数日线 + 周线（用于识别行情周期和板块级回测）
- 5 只候选个股日线 + 周线（用于个股验证）

## 行情周期识别

对证券板块指数收盘价做平滑（默认 20 周期），用 `scipy.signal.argrelextrema` 检测波峰波谷，合并过滤：
- 合并相邻同向极值，保留最显著者
- 过滤振幅 < 最小振幅阈值（默认 15%）的波动
- 过滤持续 < 最短天数（默认 20 交易日）的行情

可调参数（侧边栏滑块）：
- 平滑窗口（默认 20）
- 最小振幅阈值（默认 15%）
- 最短持续天数（默认 20）

## 技术指标库

总计约 14 个指标、30~50 种参数组合。指标聚焦**早期预警信号**而非滞后确认，核心策略：用背离替代交叉、用拐头替代突破、用缩量止跌替代放量突破。

### 趋势类（3 个）

| 指标 | 信号逻辑 | 参数搜索范围 |
|------|---------|------------|
| MACD 底背离 | 价格新低但 DIF 未新低，或柱线连续 N 日缩短 | 快线 [5,12]、慢线 [26,34]、信号线 [5,9] |
| MA 均线回归 | 价格在长期均线下运行 N 日后首次站上均线 | 长周期 [60, 120]、线下天数 [10,20,30] |
| ADX 低位拐头 | ADX 在低位（<阈值）开始上升，且 +DI 上穿 -DI | 周期 [14,21]、低位阈值 [15,20,25] |

### 动量类（3 个）

| 指标 | 信号逻辑 | 参数搜索范围 |
|------|---------|------------|
| RSI 底背离 | 价格新低但 RSI 未新低；或 RSI 在超卖区形成 W 底 | 周期 [6,14]、超卖阈值 [25,30,35] |
| KDJ J 值拐头 | J 值 <0 后首次上穿阈值 | 周期 [9,3,3]、J 上穿阈值 [0,10,20] |
| CCI 反转 | CCI 从 < -100 区域回升上穿 -100 | 周期 [14,20] |

### 成交量类（3 个）

| 指标 | 信号逻辑 | 参数搜索范围 |
|------|---------|------------|
| 缩量止跌 | 成交量缩至 N 日均量的一定比例以下 + 价格横盘 | 缩量比例 [0.3,0.5]、横盘天数 [3,5,10] |
| 温和放量 | 缩量后首次温和放量（倍量） + 收阳线 | 倍量阈值 [1.3,1.5,1.8]、缩量参考天数 [5,10] |
| OBV 底背离 | 价格横盘或新低但 OBV 率先上行 | — |

### 波动类（2 个）

| 指标 | 信号逻辑 | 参数搜索范围 |
|------|---------|------------|
| 布林带收窄 | 带宽创 N 日新低（预示变盘）+ 价格在低位 | 周期 [20,30]、新低窗口 [40,60,120] |
| 布林带下轨回归 | 价格沿下轨运行后首次收盘站上中轨 | 周期 [20,30] |

### 资金流向类（2 个，有数据则启用）

| 指标 | 信号逻辑 |
|------|---------|
| 主力净流入转正 | 连续 N 日主力净流入由负转正 |
| 融资余额企稳 | 融资余额从持续下降转为持平或微升 |

资金类指标取决于 tushare 是否提供对应接口（tushare 2000 积分可能无权限），若数据不可得则跳过，不影响整体流程。

### 周期维度

每个指标均生成日线版和周线版（通过 `freq` 参数区分），作为独立规则参与回测。回测结果可直观对比日线指标与周线指标的灵敏度差异。

## 回测引擎

### 信号判定规则

对于每轮行情 `(T_start, T_end)`：

- 信号触发日 S 落在 `[T_start - N, T_start]` 区间 → ✅ **命中**（N 默认 30 天，可在侧边栏调整）
- 区间内多个信号 → 取最晚触发的一个（避免重复计数）
- S 落在 T_start 之后超过 10 天 → ❌ **太晚**（不计入命中，统计时单独记录）
- 其余时段 → ⚠️ **候选假信号**

### 穷举流程

```
for 每个指标规则:
    for 每个参数组合:
        signals = []  # 该组合在全部历史上的信号日列表
        for 每轮行情:
            if S in [T_start - N, T_start]:  signals.append((行情, S, "命中"))
        统计命中率、平均提前天数、假信号率
        输出一行到 backtest_results.csv
按综合得分排序
```

### 评价指标

| 统计量 | 计算方式 |
|--------|---------|
| 命中率 | 命中行情数 / 总行情数 |
| 平均提前天数 | 命中信号到行情启动日的平均天数 |
| 假信号率 | 非行情窗口内的信号数 / 年数 |
| 综合得分 | 命中率 × 0.5 - 假信号率 × 0.3 - abs(平均提前天 - 5) / 30 × 0.2 |

得分设计偏向高命中率，惩罚过多假信号，并对提前约 5 天（及时且不闲置）给予最高分。

## Streamlit 界面

### 侧边栏
- 数据状态指示器（是否已缓存、最新数据日期）
- 「刷新数据」按钮
- 行情识别参数滑块（平滑窗口、最小振幅、最短天数）
- 回测参数：「信号提前窗口」滑块（默认 30 天）

### 标签页 1：行情周期总览
- 证券指数 K 线图（Plotly）上叠加每轮行情的高亮区域
- 下方表格：行情序号、起止日、持续天数、涨幅
- 侧边栏参数调整后图表实时更新

### 标签页 2：指标回测排行榜
- 可排序表格：指标名 | 周期 | 参数 | 命中率 | 平均提前天 | 假信号率 | 综合得分
- 点击某行展开详情：该指标在各行情周期上的局部 K 线图，标注信号触发日
- 筛选器：按指标类别（趋势/动量/成交量/波动/资金）过滤

### 标签页 3：个股验证
- 下拉选股（5 只候选个股）
- 将排行榜前 3 名指标应用到该个股，展示当前状态
- 各指标当前信号判断（已触发 / 接近触发 / 无信号）
- 底部总览面板：综合评分与倾向（偏买 / 观望 / 偏卖）
- 可选叠加月线趋势图辅助判断宏观位置（不参与回测，仅作参考）

## 错误处理与容错

- tushare API 请求失败 → 界面提示错误信息，不崩溃
- 某只个股数据缺失（如上市较晚）→ 跳过该股，在个股验证页标注
- 资金流向数据无权限 → 界面标注「数据不可用」，回测自动排除该指标
- 行情识别参数导致识别出 0 轮或 >10 轮行情 → 界面警告提示调整参数

## 实现顺序

1. `indicators.py` — 所有技术指标计算函数
2. `app.py` — 按模块：数据拉取与缓存 → 行情周期识别 → 回测引擎 → Streamlit 界面
