# 卖出信号系统 — 实施计划（实现级）

> 买入系统解决"什么时候买"，本计划补齐"什么时候卖"。
> 设计原则：卖出信号是纯市场择时信号，和买入信号对称——只读市场状态，不管仓位。
> 每个阈值必须有据可循：镜像买入侧、TA-Lib 标准参数、或 A 股市场约定。

---

## 阶段 1：卖出指标函数 (`indicators.py`)

### 1.0 设计原则

| 原则 | 说明 |
|------|------|
| **镜像对称** | 每个卖出指标对应一个买入指标，算法结构镜像，参数取值一致或有明确理由不同 |
| **阈值有据** | 每个参数值必须给出依据：买入侧同值 / 原始论文定义 / A 股约定 |
| **TA-Lib 复用** | 使用与买入侧完全相同的 TA-Lib 函数和标准参数 |
| **注册表独立** | `SELL_INDICATOR_REGISTRY` 与 `INDICATOR_REGISTRY` 并列，不混用 |

### 1.1 趋势类

#### 1.1.1 MACD顶背离

**买入对标**：`macd_divergence` (indicators.py:27)
**函数签名**：`def macd_top_divergence(df, fast=12, slow=26, signal=9, lookback=60)`

**算法**（镜像 `macd_divergence`，troughs → peaks，lower low → higher high，DIF higher low → DIF lower high）：
```
1. close = df['close'].values
2. dif, dea, hist = talib.MACD(close, fastperiod=fast, slowperiod=slow, signalperiod=signal)
3. sig = pd.Series(False, index=df.index)
4. order = max(lookback // 4, 5)
5. peaks = argrelextrema(close, np.greater, order=order)[0]   ← 买入侧用 np.less
6. for j in range(1, len(peaks)):
     t1, t2 = peaks[j-1], peaks[j]
     跳过 NaN
     if close[t2] <= close[t1]: continue        ← 买入侧: close[t2] >= close[t1] (要 higher high)
     if dif[t2] < dif[t1]:                        ← 买入侧: dif[t2] > dif[t1] (要 lower high)
         sig.iloc[t2] = True
7. return sig
```

**参数网格**：

| 参数 | 值 | 依据 |
|------|-----|------|
| `fast` | 12 | MACD 标准参数（Appel 原始定义），与买入侧相同 |
| `slow` | 26 | MACD 标准参数，与买入侧相同 |
| `signal` | 9 | MACD 标准参数，与买入侧相同 |
| `lookback` | 60, 90 | 与买入侧 `macd_divergence` 完全一致；60≈3 交易月，90≈4.5 交易月 |

**边界处理**：TA-Lib MACD 前 slow-1=25 天返回 NaN，`np.isnan(dif[t])` 跳过，与买入侧相同。

#### 1.1.2 MACD红柱缩短

**买入对标**：`macd_histogram_shrink` (indicators.py:61)
**函数签名**：`def macd_histogram_shrink_sell(df, fast=12, slow=26, signal=9, consecutive=3)`

**算法**（镜像 `macd_histogram_shrink`，below zero → above zero，rising → falling）：
```
1. close = df['close'].values
2. dif, dea, hist = talib.MACD(close, fastperiod=fast, slowperiod=slow, signalperiod=signal)
3. sig = pd.Series(False, index=df.index)
4. for i in range(consecutive, n):
     检查 hist[i-consecutive+1 : i+1] 无 NaN
     above_zero = all(hist[窗口] >= 0)            ← 买入侧: all(<= 0)
     falling = hist 窗口内逐日递减                  ← 买入侧: 逐日递增(rising)
     if above_zero and falling:
         sig.iloc[i] = True
5. return sig
```

**参数网格**：

| 参数 | 值 | 依据 |
|------|-----|------|
| `fast, slow, signal` | 12, 26, 9 | MACD 标准参数，与买入侧相同 |
| `consecutive` | 2, 3, 5 | 与买入侧 `macd_histogram_shrink` 完全一致 |

#### 1.1.3 均线跌破

**买入对标**：`ma_return` (indicators.py:94)
**函数签名**：`def ma_breakdown(df, ma_period=60, above_days=20)`

**算法**（镜像 `ma_return`，below → above，crosses above → crosses below）：
```
1. close = df['close'].values
2. ma = talib.SMA(close, timeperiod=ma_period)
3. above = close > pd.Series(ma, index=df.index)    ← 买入侧: close < ma
4. consec = above.astype(int).groupby((above != above.shift()).cumsum()).cumsum()
5. for i in range(1, n):
     if np.isnan(ma[i]): continue
     if consec.iloc[i-1] >= above_days and above.iloc[i-1] and not above.iloc[i]:
         sig.iloc[i] = True                           ← 买入侧: below[i-1] and not below[i]
6. return sig
```

**参数网格**：

| 参数 | 值 | 依据 |
|------|-----|------|
| `ma_period` | 60, 120 | 与买入侧 `ma_return` 相同；MA60≈季线，MA120≈半年线 |
| `above_days` | 20, 30 | 镜像买入侧 `below_days=20, 30`；20≈1 交易月，30≈1.5 交易月 |

#### 1.1.4 ADX高位回落

**买入对标**：`adx_turn` (indicators.py:113)
**函数签名**：`def adx_peak_turn(df, period=14, adx_high=35)`

**算法**（镜像 `adx_turn`，low → high，turns up → turns down，+DI cross above → +DI cross below）：
```
1. adx = talib.ADX(high, low, close, timeperiod=period)
2. p_di = talib.PLUS_DI(high, low, close, timeperiod=period)
3. m_di = talib.MINUS_DI(high, low, close, timeperiod=period)
4. for i in range(period*2, n):
     跳过 NaN
     if (adx[i] < adx[i-1] and                    ← 买入侧: adx[i] > adx[i-1] (turns up)
         adx[i-1] >= adx_high and                  ← 买入侧: adx[i-1] <= adx_low (from low)
         p_di[i] < m_di[i] and                     ← 买入侧: p_di[i] > m_di[i] (+DI cross above)
         p_di[i-1] >= m_di[i-1]):                  ← 买入侧: p_di[i-1] <= m_di[i-1]
         sig.iloc[i] = True
5. return sig
```

**参数网格**：

| 参数 | 值 | 依据 |
|------|-----|------|
| `period` | 14 | Wilder 原始 ADX 周期，与买入侧相同 |
| `adx_high` | 35, 40 | Wilder 分类：ADX>25=强趋势，>35=极强，>40=极端。买入侧用 20/25（弱趋势启动），卖出侧用 35/40（强趋势衰竭），对称设计 |

### 1.2 动量类

#### 1.2.1 RSI顶背离

**买入对标**：`rsi_divergence` (indicators.py:141)
**函数签名**：`def rsi_top_divergence(df, period=14, lookback=60)`

**算法**（镜像 `rsi_divergence`，new low → new high，> min*0.95 → < max*1.05，< 50 → > 50）：
```
1. close = df['close'].values
2. rsi = talib.RSI(close, timeperiod=period)
3. for i in range(lookback, n):
     wc = close[i-lookback:i+1]    # 价格窗口
     wr = rsi[i-lookback:i+1]      # RSI 窗口
     wr = wr[~NaN]
     if (wc[-1] >= wc.max() and              ← 买入侧: wc[-1] <= wc.min() (价格新低→新高)
         rsi[i] < wr.max() * 1.05 and         ← 买入侧: rsi[i] > wr.min() * 0.95
         rsi[i] > 50):                        ← 买入侧: rsi[i] < 50
         sig.iloc[i] = True
4. return sig
```

**参数网格**：

| 参数 | 值 | 依据 |
|------|-----|------|
| `period` | 14 | Wilder 原始 RSI 周期，与买入侧相同 |
| `lookback` | 60, 90 | 与买入侧 `rsi_divergence` 完全一致 |
| 系数 1.05 | — | 镜像买入侧 0.95：5% 噪声容差。买入侧允许 RSI 略低于前低（×0.95），卖出侧允许 RSI 略高于前高（×1.05） |
| 阈值 50 | — | RSI 中轴，标准多空分界线。买入侧 < 50（空头区背离），卖出侧 > 50（多头区背离） |

#### 1.2.2 KDJ高位死叉

**买入对标**：`kdj_j_reversal` (indicators.py:165)
**函数签名**：`def kdj_death_cross(df, n=9, m1=3, m2=3)`

**算法**（镜像 `kdj_j_reversal`，< 0 → > 100，crosses above 20 → crosses below 80）：
```
1. k, d = talib.STOCH(high, low, close, fastk_period=n, slowk_period=m1, slowd_period=m2)
2. j = 3*k - 2*d
3. for i in range(2, n):
     跳过 NaN
     if j[i-1] > 100 and j[i] < 80:       ← 买入侧: j[i-1] < 0 and j[i] > 20
         sig.iloc[i] = True
4. return sig
```

**参数网格**：

| 参数 | 值 | 依据 |
|------|-----|------|
| `n, m1, m2` | 9, 3, 3 | KDJ 标准参数（Lane 原始定义），与买入侧相同 |
| 阈值 100 | — | J 值无上界，> 100 为极端超买。镜像买入侧 < 0（极端超卖） |
| 阈值 80 | — | J 从 > 100 回落到 80 以下 = 超买区退出。镜像买入侧：J 从 < 0 反弹到 20 以上 = 超卖区退出。0/20 和 100/80 是 KDJ 标准 overbought/oversold 区间 |

### 1.3 成交量类

#### 1.3.1 放量滞涨

**买入对标**：`volume_contraction` (indicators.py:210) 的反向逻辑
**函数签名**：`def volume_stagnation(df, ma_period=20, vol_ratio=1.3, stagnant_days=3, chg_pct=1.0)`

**算法**：
```
1. vol = df['vol'].values
2. close = df['close'].values
3. vol_ma = talib.SMA(vol, timeperiod=ma_period)
4. for i in range(stagnant_days, n):
     跳过 NaN
     检查窗口 [i-stagnant_days+1, i] 内每一天 idx：
       vol[idx] >= vol_ma[idx] * vol_ratio          ← 放量（买入侧 volume_contraction: vol < ma*0.5 缩量）
       abs((close[idx]-close[idx-1])/close[idx-1]*100) <= chg_pct  ← 涨跌幅小
     若窗口内每天都满足 → sig.iloc[i] = True
5. return sig
```

**参数网格**：

| 参数 | 值 | 依据 |
|------|-----|------|
| `ma_period` | 20 | 与 `volume_contraction` 相同，20 是成交量均量线标准周期 |
| `vol_ratio` | 1.3 | 与 `moderate_expansion` 的 `expand_ratio=1.3` 相同；1.3 倍均量是"放量"的常见门槛 |
| `stagnant_days` | 3, 5 | 镜像 `macd_histogram_shrink` 的 `consecutive=3, 5`；3 天最小模式，5 天更显著 |
| `chg_pct` | 1.0 | `moderate_expansion` 用 1.0% 作为最小涨幅门槛；放量但涨幅 < 1.0% = 滞涨（放量不涨 = 派发） |

#### 1.3.2 缩量新高

**买入对标**：无直接对应（`moderate_expansion` 检查放量，此指标检查缩量，是派发特征）
**函数签名**：`def shrink_new_high(df, newhigh_days=20, shrink_days=3)`

**算法**：
```
1. close = df['close'].values
2. vol = df['vol'].values
3. vol_ma = talib.SMA(vol, timeperiod=20)
4. for i in range(max(newhigh_days, shrink_days), n):
     跳过 NaN
     # 条件1：价格创 newhigh_days 日新高
     if close[i] < np.max(close[i-newhigh_days:i]): continue
     # 条件2：成交量连续 shrink_days 天递减
     vol_window = vol[i-shrink_days+1 : i+1]
     if all(vol_window[j] < vol_window[j-1] for j in range(1, len(vol_window))):
         sig.iloc[i] = True
5. return sig
```

**参数网格**：

| 参数 | 值 | 依据 |
|------|-----|------|
| `newhigh_days` | 20, 40 | 20≈1 交易月，40≈2 交易月；比 `bollinger_squeeze` 的 40/60/120 短，因为检测的是近期量价背离而非长期形态 |
| `shrink_days` | 3, 5 | 与其他指标的 `consecutive=3, 5` 一致；3 天最小模式，5 天更显著 |

#### 1.3.3 OBV顶背离

**买入对标**：`obv_divergence` (indicators.py:254)
**函数签名**：`def obv_top_divergence(df, lookback=60)`

**算法**（镜像 `obv_divergence`，price below mean → above mean，OBV above mean → below mean）：
```
1. obv = talib.OBV(close, vol)

     if wc[-1] >= wc.mean() and obv[i] < wo.mean():    ← 买入侧: wc[-1] <= wc.mean() and obv[i] > wo.mean()
         sig.iloc[i] = True
3. return sig
```

**参数网格**：

| 参数 | 值 | 依据 |
|------|-----|------|
| `lookback` | 60, 90 | 与买入侧 `obv_divergence` 完全一致 |

### 1.4 波动类

#### 1.4.1 布林带上轨触顶

**买入对标**：`bollinger_lower_rebound` (indicators.py:295)
**函数签名**：`def bollinger_upper_touch(df, period=20, std=2)`

**算法**（镜像 `bollinger_lower_rebound`，lower → upper，crosses above middle → crosses below middle）：
```
1. upper, middle, lower = talib.BBANDS(close, timeperiod=period, nbdevup=std, nbdevdn=std)
2. high = df['high'].values
3. for i in range(period+3, n):
     跳过 NaN
     near_upper = high[i-3:i+1].max() >= upper[i-3:i+1].min()    ← 买入侧: low[i-3:i+1].min() <= lower[i-3:i+1].max()
     if (close[i-1] > middle[i-1] and close[i] < middle[i]       ← 买入侧: close[i-1] < middle[i-1] and close[i] > middle[i]
         and near_upper):
         sig.iloc[i] = True
4. return sig
```

**参数网格**：

| 参数 | 值 | 依据 |
|------|-----|------|
| `period` | 20 | Bollinger 原始周期，与买入侧相同 |
| `std` | 2 | Bollinger 原始标准差倍数，与买入侧相同 |

#### 1.4.2 布林带宽极值回落

**买入对标**：`bollinger_squeeze` (indicators.py:279)
**函数签名**：`def bollinger_width_climax(df, period=20, std=2, ndays_newhigh=60)`

**算法**（镜像 `bollinger_squeeze`，N-day low → N-day high）：
```
1. upper, middle, lower = talib.BBANDS(close, timeperiod=period, nbdevup=std, nbdevdn=std)
2. bandwidth = (upper - lower) / (middle + 1e-10)
3. for i in range(ndays_newhigh, n):
     跳过 NaN
     if bandwidth[i] >= np.nanmax(bandwidth[i-ndays_newhigh:i]):    ← 买入侧: <= np.nanmin(...)
         sig.iloc[i] = True
4. return sig
```

**参数网格**：

| 参数 | 值 | 依据 |
|------|-----|------|
| `period` | 20 | Bollinger 原始周期，与买入侧相同 |
| `std` | 2 | Bollinger 原始标准差倍数，与买入侧相同 |
| `ndays_newhigh` | 60, 120 | 买入侧 `bollinger_squeeze` 用 40/60/120；卖出侧用 60/120（去掉 40），因为波动率冲顶需要更长时间窗口才有意义，40 天太短无法区分真正的极值 |

### 1.5 资金类

#### 1.5.1 主力连续净流出

**买入对标**：`main_force_inflow` (indicators.py:317)
**函数签名**：`def main_force_outflow(df, consecutive_days=3)`

**算法**（镜像 `main_force_inflow`，net > 0 → net < 0）：
```
1. if 'buy_elg_vol' not in df.columns or 'sell_elg_vol' not in df.columns:
     return empty sig
2. net = df['buy_elg_vol'] - df['sell_elg_vol']
3. negative = net < 0                              ← 买入侧: positive = net > 0
4. for i in range(consecutive_days-1, n):
     if negative.iloc[i-consecutive_days+1 : i+1].all():
         sig.iloc[i] = True
5. return sig
```

**参数网格**：

| 参数 | 值 | 依据 |
|------|-----|------|
| `consecutive_days` | 3 | 与买入侧 `main_force_inflow` 完全一致 |

#### 1.5.2 融资余额衰减

**买入对标**：`margin_stabilize` (indicators.py:329)
**函数签名**：`def margin_decline(df, window=5, threshold=-1.0)`

**算法**（镜像 `margin_stabilize`，> threshold → < threshold）：
```
1. if 'rzye' not in df.columns: return empty sig

```

**参数网格**：

### 1.9 `filter_sell_signals()`

**买入对标**：`filter_signals` (indicators.py:484)
**函数签名**：
```python
def filter_sell_signals(df, signal_series, ma_period=MA_PERIOD, rise_pct=RISE_PCT,
                        cooldown=SIGNAL_COOLDOWN, state_window=STATE_WINDOW,
                        ref_lows=None, price_surge_threshold=PRICE_DROP_THRESHOLD):
```

**与 `filter_signals` 的镜像对照**：

| 买入侧 `filter_signals` | 卖出侧 `filter_sell_signals` | 依据 |
|--------------------------|------------------------------|------|
| `close < MA250` | `close > MA250` | 趋势方向反转：买入只在年线下（熊市），卖出只在年线上（牛市） |
| `decline > DECLINE_PCT(20%)` | `rise > RISE_PCT(20%)` | 跌幅→涨幅，同一 20% 门槛 |
| `ref_highs`（周期峰价） | `ref_lows`（周期谷价） | 参考点反转 |
| `decline = (ref_high - close) / ref_high * 100` | `rise = (close - ref_low) / ref_low * 100` | 跌幅→涨幅公式 |
| `SIGNAL_COOLDOWN = 15` | `SIGNAL_COOLDOWN = 15` | 复用，同一市场同一交易节奏 |
| `STATE_WINDOW = 5` | `STATE_WINDOW = 5` | 复用，同一 latching 逻辑 |
| 价格跌破逃逸：close < last_signal_close * (1 - 2%) | 价格飙升逃逸：close > last_signal_close * (1 + 2%) | 镜像：买入侧允许更低价二次入场，卖出侧允许更高价二次出场 |

**算法**：
```python
def filter_sell_signals(df, signal_series, ma_period=MA_PERIOD, rise_pct=RISE_PCT,
                        cooldown=SIGNAL_COOLDOWN, state_window=STATE_WINDOW,
                        ref_lows=None, price_surge_threshold=PRICE_DROP_THRESHOLD):
    close = df['close'].values.astype(float)
    n = len(close)
    ma = talib.SMA(close, timeperiod=ma_period)

    # 计算涨幅（镜像买入侧的跌幅）
    if ref_lows is not None:
        rise = (close - np.array(ref_lows)) / (np.array(ref_lows) + 1e-10) * 100
    else:
        rolling_low = pd.Series(close).rolling(ma_period, min_periods=1).min().values
        rise = (close - rolling_low) / (rolling_low + 1e-10) * 100

    # 上下文：市场处于上涨模式（close > MA250 且 涨幅 > 20%）
    context_ok = np.zeros(n, dtype=bool)
    for i in range(n):
        if not np.isnan(ma[i]) and not np.isnan(rise[i]):
            context_ok[i] = (close[i] > ma[i]) and (rise[i] > rise_pct)

    # 冷却 + 状态窗口（与 filter_signals 完全相同的单遍扫描逻辑）
    filtered = signal_series.copy()
    last_context_idx = -state_window - 1
    last_signal_idx = -cooldown - 1
    signal_set = set(np.where(signal_series.values)[0])

    for i in range(n):
        if context_ok[i]:
            last_context_idx = i
        if i in signal_set:
            if np.isnan(ma[i]):
                filtered.iloc[i] = False
            elif i - last_context_idx > state_window:
                filtered.iloc[i] = False
            elif i - last_signal_idx < cooldown:
                # 价格飙升逃逸：cooldown 内价格飙升 > threshold% 则放行
                if (price_surge_threshold is not None
                        and last_signal_idx >= 0
                        and close[i] > close[last_signal_idx] * (1 + price_surge_threshold / 100.0)):
                    last_signal_idx = i
                else:
                    filtered.iloc[i] = False
            else:
                last_signal_idx = i

    return filtered
```

```python
def _build_stock_ref_lows(stock_df, stock_code):
    # 周期检测与 _build_stock_ref_highs 完全相同（同一套 troughs/peaks）
    # 区别：用 cycle troughs（start_date, start_price）构建 ref_lows
    cache_key = (stock_code, len(stock_df), 'low')  # 加 'low' 后缀避免与 ref_highs 缓存冲突
    if cache_key in _ref_lows_cache:
        return _ref_lows_cache[cache_key]

    # ... 周期检测逻辑与 _build_stock_ref_highs 完全相同 ...

    first_cycle_start = cycle_df['start_date'].min()
    close_vals = stock_df['close'].values.astype(float)
    rolling_min = pd.Series(close_vals).rolling(MA_PERIOD, min_periods=1).min().values
    cycle_troughs_sorted = sorted([(c['start_date'], c['start_price']) for _, c in cycle_df.iterrows()])

    ref_lows = []
    for i, d in enumerate(stock_df.index):
        if d < first_cycle_start:

                else: break

                if d > last_cycle_start:
                    rl = min(rl, rolling_min[i])       # 买入侧: max(rh, rolling_max)
                ref_lows.append(rl)
            else:
                ref_lows.append(rolling_min[i])

    _ref_lows_cache[cache_key] = ref_lows
    return ref_lows
```

## 阶段 2：卖出信号回测引擎 (`backtest.py`)

### 2.0 实际最高价锚点

**问题**：`cycles.csv` 的 `end_date`/`end_price` 是平滑线上的峰，非实际最高收盘价。
**解决**：回测前对每轮周期计算实际最高收盘价和对应日期：

```python
def _compute_actual_peaks(df, cycles):
    """对每轮周期计算实际最高收盘价（非平滑峰）。"""
    for cycle in cycles:
        start = pd.Timestamp(cycle['start_date'])
        end = pd.Timestamp(cycle['end_date'])
        mask = (df.index >= start) & (df.index <= end)
        cycle_data = df[mask]
        if not cycle_data.empty:
            cycle['actual_peak_price'] = cycle_data['close'].max()
            cycle['actual_peak_date'] = cycle_data['close'].idxmax()
        else:
            cycle['actual_peak_price'] = cycle.get('end_price', 0)
            cycle['actual_peak_date'] = end
    return cycles
```

**后续所有评估使用 `actual_peak_price` / `actual_peak_date`，不用 `end_price` / `end_date`。**

### 2.1 `judge_sell_signal()`

**买入对标**：`judge_signal` (backtest.py:41)
**函数签名**：
```python
def judge_sell_signal(signal_series, actual_peak_date, actual_peak_price, close_series,
                      window_before=SELL_PEAK_BEFORE, window_after=SELL_PEAK_AFTER):
```

**与 `judge_signal` 的镜像对照**：

| 买入侧 `judge_signal` | 卖出侧 `judge_sell_signal` | 依据 |
|------------------------|----------------------------|------|
| 锚点：`cycle_start`（周期谷） | 锚点：`actual_peak_date`（周期峰） | 谷→峰 |
| 窗口：`[start-30, start]` | 窗口：`[peak-10, peak]` | 买入侧用 30 天（SIGNAL_WINDOW_INDEX），卖出侧用 10 天（SELL_PEAK_BEFORE）。卖出侧更紧因为卖早了代价更大 |
| 迟到窗口：`[start, start+10]` | 迟到窗口：`[peak, peak+15]` | 买入侧 LATE_CUTOFF_DAYS=10，卖出侧 SELL_PEAK_AFTER=15。卖出侧略宽因为峰后下跌时信号仍有价值 |
| 取窗口内最新信号 | 取窗口内距峰最近的信号 | 买入侧窗口全在峰前，最新=最接近谷。卖出侧窗口跨峰，需取绝对距离最小 |
| `days_before = (start - signal_date).days` | `days_from_peak = (signal_date - peak_date).days` | 负=峰前，正=峰后 |
| 无捕获率概念 | `capture_rate = signal_close / actual_peak_price` | 卖出新增：卖出价占峰价比例，衡量卖出质量 |
| 返回：hit, signal_date, days_before, late | 返回：hit, signal_date, days_from_peak, capture_rate, early, late | 新增 capture_rate, early |

**参数依据**：

| 参数 | 值 | 依据 |
|------|-----|------|
| `SELL_PEAK_BEFORE` | 10 | 买入侧 SIGNAL_WINDOW_INDEX=30；卖出侧更紧（10 天），因为卖出信号离峰太远（>10 天）意味着踏空大量涨幅。10 天 ≈ 2 交易周，是"接近顶部"的合理范围 |
| `SELL_PEAK_AFTER` | 15 | 比 PEAK_BEFORE 略宽（15>10），因为峰后下跌时卖出信号仍有实用价值（逃顶不必精确在最高点）。15 天 ≈ 3 交易周 |

**算法**：
```python
def judge_sell_signal(signal_series, actual_peak_date, actual_peak_price, close_series,
                      window_before=SELL_PEAK_BEFORE, window_after=SELL_PEAK_AFTER):
    peak = pd.Timestamp(actual_peak_date)

    # 命中窗口：[peak - window_before, peak]
    hit_start = peak - pd.Timedelta(days=window_before)
    hit_mask = (signal_series.index >= hit_start) & (signal_series.index <= peak)
    hit_signals = signal_series[hit_mask]

    # 迟到窗口：[peak, peak + window_after]
    late_end = peak + pd.Timedelta(days=window_after)
    late_mask = (signal_series.index > peak) & (signal_series.index <= late_end)
    late_signals = signal_series[late_mask]

    # 合并命中+迟到，取距峰最近的信号
    all_window_signals = signal_series[hit_mask | late_mask]
    if all_window_signals.any():
        signal_dates = all_window_signals[all_window_signals].index
        # 取距峰最近的信号
        signal_date = min(signal_dates, key=lambda d: abs((d - peak).days))
        days_from_peak = (signal_date - peak).days  # 负=峰前, 正=峰后
        signal_close = close_series.loc[signal_date]
        capture_rate = signal_close / actual_peak_price  # 0-1 之间

        is_hit = hit_mask.loc[signal_date]  # 在命中窗口内
        is_late = late_mask.loc[signal_date]  # 在迟到窗口内

        return {
            'hit': is_hit,
            'signal_date': signal_date,
            'days_from_peak': days_from_peak,
            'capture_rate': capture_rate,
            'late': is_late,
        }

    return {
        'hit': False,
        'signal_date': None,
        'days_from_peak': None,
        'capture_rate': None,
        'late': False,
    }
```

### 2.2 `count_false_sell_signals()`

**买入对标**：`count_false_signals` (backtest.py:91)
**算法**（镜像：start → actual_peak_date，SIGNAL_WINDOW_INDEX → SELL_PEAK_BEFORE，LATE_CUTOFF_DAYS → SELL_PEAK_AFTER）：
```python
def count_false_sell_signals(signal_series, cycles, window_before=SELL_PEAK_BEFORE,
                              window_after=SELL_PEAK_AFTER):
    valid_mask = pd.Series(False, index=signal_series.index)
    for cycle in cycles:
        peak = pd.Timestamp(cycle['actual_peak_date'])
        window_start = peak - pd.Timedelta(days=window_before)
        window_end = peak + pd.Timedelta(days=window_after)
        valid_mask[(signal_series.index >= window_start) & (signal_series.index <= window_end)] = True
    false_signals = signal_series & ~valid_mask
    return false_signals.sum()
```

### 2.3 `run_sell_backtest()`

**买入对标**：`run_backtest` (backtest.py:109)
**函数签名**：
```python
def run_sell_backtest(cycle_df, sell_signals_list, data_dfs,
                      window_before=SELL_PEAK_BEFORE, window_after=SELL_PEAK_AFTER):
```

**与 `run_backtest` 的结构对照**：

| 买入侧 `run_backtest` | 卖出侧 `run_sell_backtest` |
|------------------------|----------------------------|
| `ref_highs` 构建 | `ref_lows` 构建 |
| `filter_signals` | `filter_sell_signals` |
| `judge_signal` | `judge_sell_signal` |
| `count_false_signals` | `count_false_sell_signals` |
| 评分：命中率×0.5 + 精度×0.2 - 天数扣分×0.3 | 评分：捕获率×0.5 + 命中率×0.3 + 精度×0.2 |

**评分公式**：

```python
score = avg_capture * WEIGHT_SELL_CAPTURE      # 0.5
      + weighted_hr * WEIGHT_SELL_HIT_RATE      # 0.3
      + precision * WEIGHT_SELL_PRECISION        # 0.2
```

| 权重 | 值 | 依据 |
|------|-----|------|
| `WEIGHT_SELL_CAPTURE` | 0.5 | 捕获率是卖出信号最重要的质量指标（卖出价/峰价）。买入侧最重要的命中率(0.5)在卖出侧让位给捕获率，因为卖出质量用价格衡量比用"是否触发"更精确 |
| `WEIGHT_SELL_HIT_RATE` | 0.3 | 命中率（信号是否在峰窗口内触发）退居第二。买入侧命中率权重 0.5，卖出侧降为 0.3，因为捕获率已包含时序信息 |
| `WEIGHT_SELL_PRECISION` | 0.2 | 信号有效率（窗口内信号/全部信号），与买入侧相同权重 0.2 |
| 无天数扣分 | — | 买入侧用 days_score 惩罚偏离理想天数。卖出侧不需要，因为 capture_rate 已经惩罚了偏离峰值的信号（卖得远→价格低→捕获率低→分数低）。capture_rate 是价格维度，比天数维度更精确 |

**最大得分**：0.5 + 0.3 + 0.2 = 1.0（买入侧最大 0.7，两侧分值不直接可比但各自内部排序有效）

**算法**：
```python
def run_sell_backtest(cycle_df, sell_signals_list, data_dfs,
                      window_before=SELL_PEAK_BEFORE, window_after=SELL_PEAK_AFTER):
    from indicators import filter_sell_signals

    cycles_all = cycle_df.to_dict('records')
    cycles = [c for c in cycles_all if pd.Timestamp(c['start_date']) >= pd.Timestamp(CYCLE_FILTER_DATE)]
    total_cycles = len(cycles)
    cycle_weights = np.array([c['change_pct'] / 100 for c in cycles])
    total_weight = cycle_weights.sum()

    results = []

    for freq_key, df in [('日线', data_dfs.get('index_daily')),
                          ('周线', data_dfs.get('index_weekly'))]:
        if df is None or df.empty:
            continue
        df = df.set_index('trade_date').sort_index()

        # 构建实际最高价锚点
        cycles = _compute_actual_peaks(df, cycles)

        # 构建 ref_lows（镜像买入侧的 ref_highs）
        cycle_troughs = sorted([(pd.Timestamp(c['start_date']), c['start_price']) for c in cycles_all])
        ref_lows = []
        for d in df.index:
            rl = None
            for trough_date, trough_price in cycle_troughs:
                if trough_date <= d:
                    rl = trough_price
                else:
                    break
            ref_lows.append(rl if rl is not None else df.loc[d, 'close'])

        close_series = df['close']

        for rule in sell_signals_list:
            work_df = df.copy()
            # moneyflow 合并（与买入侧相同）
            if rule.get('requires_moneyflow', False):
                mf_df = data_dfs.get('index_moneyflow')
                if mf_df is None or mf_df.empty:
                    continue
                mf_df = mf_df.set_index('trade_date').sort_index()
                for col in mf_df.columns:
                    if col not in work_df.columns:
                        work_df[col] = mf_df[col]

            try:
                signal_series = rule['func'](work_df.reset_index(), **rule['params'])
                signal_series.index = work_df.index
                signal_series = filter_sell_signals(
                    work_df.reset_index(), signal_series, ref_lows=ref_lows,
                )
                signal_series.index = work_df.index
            except Exception as e:
                print(f"  [SKIP] {rule['name']} ({freq_key}): {e}")
                continue

            total_signals = int(signal_series.sum())
            hits = 0
            lates = 0
            capture_list = []
            days_list = []
            hit_signal_count = 0
            weighted_hits = 0

            for i, cycle in enumerate(cycles):
                peak_date = pd.Timestamp(cycle['actual_peak_date'])
                peak_price = cycle['actual_peak_price']

                # 精度统计：窗口内是否有信号
                ws = peak_date - pd.Timedelta(days=window_before)
                we = peak_date + pd.Timedelta(days=window_after)
                wm = (signal_series.index >= ws) & (signal_series.index <= we)
                if signal_series[wm].any():
                    hit_signal_count += 1

                judgment = judge_sell_signal(
                    signal_series, peak_date, peak_price, close_series,
                    window_before, window_after,
                )
                if judgment['hit']:
                    hits += 1
                    capture_list.append(judgment['capture_rate'])
                    days_list.append(judgment['days_from_peak'])
                    weighted_hits += cycle_weights[i]
                elif judgment['late']:
                    lates += 1
                    capture_list.append(judgment['capture_rate'])
                    days_list.append(judgment['days_from_peak'])

            hit_rate = hits / total_cycles if total_cycles > 0 else 0
            weighted_hr = weighted_hits / total_weight if total_weight > 0 else 0
            avg_capture = np.mean(capture_list) if capture_list else 0
            avg_days = np.mean(days_list) if days_list else None
            precision = hit_signal_count / total_signals if total_signals > 0 else 0

            score = (avg_capture * WEIGHT_SELL_CAPTURE
                     + weighted_hr * WEIGHT_SELL_HIT_RATE
                     + precision * WEIGHT_SELL_PRECISION)

            results.append({
                '指标名': rule['name'],
                '类别': rule['category'],
                '周期': freq_key,
                '参数': _param_str(rule['params']),
                '命中率': round(weighted_hr, 3),
                '命中轮数': f'{hits}/{total_cycles}',
                '平均捕获率': round(avg_capture, 3),
                '平均距峰天': round(avg_days, 1) if avg_days is not None else None,
                '信号有效率': round(precision, 3),
                '总信号数': total_signals,
                '综合得分': round(score, 3),
            })
```

## 阶段 3：买入-卖出联合回测 (`backtest.py`)

### 3.1 配对规则

| 场景 | 规则 | 依据 |
|------|------|------|
| 同一轮多个买入信号触发 | 取 Top 1 买入指标（按买入回测得分排名）的信号 | 与现有 `judge_signal` 取最新信号一致 |
| 同一轮多个卖出信号触发 | 取 Top 1 卖出指标（按卖出回测得分排名）的信号 | 对称设计 |
| 买入信号触发但无卖出信号 | 标记"未退出"，持有到周期结束 | 信号系统只负责检测，不强制止损 |
| 卖出信号触发但无买入信号 | 跳过该轮 | 无法模拟没有入场的交易 |
| 买入后卖出信号在同周期内触发 | 正常配对，计算收益 | — |
| 买入后卖出信号在下一周期才触发 | 跨周期配对，但归属到买入所在周期 | 以买入信号所在周期为锚 |

**算法**：
```python
def run_combined_backtest(cycle_df, buy_results, sell_results, index_daily,
                          signal_window=SIGNAL_WINDOW_INDEX,
                          window_before=SELL_PEAK_BEFORE,
                          window_after=SELL_PEAK_AFTER):
    from indicators import INDICATOR_REGISTRY, SELL_INDICATOR_REGISTRY, filter_signals, filter_sell_signals

    cycles_all = cycle_df.to_dict('records')
    cycles = [c for c in cycles_all if pd.Timestamp(c['start_date']) >= pd.Timestamp(CYCLE_FILTER_DATE)]

    df = index_daily.set_index('trade_date').sort_index()
    cycles = _compute_actual_peaks(df, cycles)
    close_series = df['close']

    # 取买入/卖出 Top 1 指标（按得分排名）
    buy_top = buy_results.iloc[0]  # 得分最高的买入指标
    sell_top = sell_results.iloc[0]  # 得分最高的卖出指标
    buy_name = buy_top['指标名']
    sell_name = sell_top['指标名']

    buy_cfg = INDICATOR_REGISTRY.get(buy_name)
    sell_cfg = SELL_INDICATOR_REGISTRY.get(sell_name)
    if buy_cfg is None or sell_cfg is None:
        return pd.DataFrame()

    # 构建买入侧 ref_highs（现有逻辑）
    cycle_peaks = sorted([(pd.Timestamp(c['end_date']), c['end_price']) for c in cycles_all])
    ref_highs = []
    for d in df.index:
        rh = None
        for peak_date, peak_price in cycle_peaks:
            if peak_date <= d: rh = peak_price
            else: break
        ref_highs.append(rh if rh is not None else df.loc[d, 'close'])

    # 构建卖出侧 ref_lows
    cycle_troughs = sorted([(pd.Timestamp(c['start_date']), c['start_price']) for c in cycles_all])
    ref_lows = []
    for d in df.index:
        rl = None
        for trough_date, trough_price in cycle_troughs:
            if trough_date <= d: rl = trough_price
            else: break
        ref_lows.append(rl if rl is not None else df.loc[d, 'close'])

    # 计算买入信号
    buy_sig = buy_cfg['func'](df.reset_index(), **buy_cfg['params'][0])
    buy_sig.index = df.index
    buy_sig = filter_signals(df.reset_index(), buy_sig, ref_highs=ref_highs)
    buy_sig.index = df.index

    # 计算卖出信号
    sell_sig = sell_cfg['func'](df.reset_index(), **sell_cfg['params'][0])
    sell_sig.index = df.index
    sell_sig = filter_sell_signals(df.reset_index(), sell_sig, ref_lows=ref_lows)
    sell_sig.index = df.index

    # 逐周期配对
    results = []
    for i, cycle in enumerate(cycles):
        start = pd.Timestamp(cycle['start_date'])
        peak_date = pd.Timestamp(cycle['actual_peak_date'])
        peak_price = cycle['actual_peak_price']
        end = pd.Timestamp(cycle['end_date'])

        # 买入信号：取 [start - signal_window, start + LATE_CUTOFF_DAYS] 内最新
        buy_ws = start - pd.Timedelta(days=signal_window)
        buy_we = start + pd.Timedelta(days=LATE_CUTOFF_DAYS)
        buy_wm = (buy_sig.index >= buy_ws) & (buy_sig.index <= buy_we)
        buy_signals = buy_sig[buy_wm]
        buy_signal_date = buy_signals[buy_signals].index[-1] if buy_signals.any() else None

        # 卖出信号：取买入信号后、周期结束前，距峰最近的
        if buy_signal_date is not None:
            sell_wm = (sell_sig.index > buy_signal_date) & (sell_sig.index <= end)
            sell_signals = sell_sig[sell_wm]
            if sell_signals.any():
                sell_dates = sell_signals[sell_signals].index
                sell_signal_date = min(sell_dates, key=lambda d: abs((d - peak_date).days))
            else:
                sell_signal_date = None  # 未退出
        else:
            sell_signal_date = None

        # 计算指标
        if buy_signal_date is not None and sell_signal_date is not None:
            buy_price = close_series.loc[buy_signal_date]
            sell_price = close_series.loc[sell_signal_date]
            holding_days = (sell_signal_date - buy_signal_date).days
            actual_return = (sell_price - buy_price) / buy_price * 100
            max_return = (peak_price - buy_price) / buy_price * 100
            combined_capture = actual_return / max_return * 100 if max_return > 0 else 0
            rating = '★★★' if combined_capture > 80 else '★★' if combined_capture > 50 else '★' if combined_capture > 0 else '✗'
        elif buy_signal_date is not None:
            buy_price = close_series.loc[buy_signal_date]
            sell_price = close_series.loc[end]
            holding_days = (end - buy_signal_date).days
            actual_return = (sell_price - buy_price) / buy_price * 100
            max_return = (peak_price - buy_price) / buy_price * 100
            combined_capture = actual_return / max_return * 100 if max_return > 0 else 0
            rating = '未退出'
        else:
            continue  # 无买入信号，跳过

        results.append({
            '周期': i + 1,
            '起始日': start.strftime('%Y-%m-%d'),
            '峰值日': peak_date.strftime('%Y-%m-%d'),
            '买入指标': buy_name,
            '买入日': buy_signal_date.strftime('%Y-%m-%d'),
            '买入价': round(buy_price, 2),
            '卖出指标': sell_name if sell_signal_date else '—',
            '卖出日': sell_signal_date.strftime('%Y-%m-%d') if sell_signal_date else '持有至周期结束',
            '卖出价': round(sell_price, 2),
            '持有天数': holding_days,
            '实际收益率%': round(actual_return, 1),
            '最大可获收益率%': round(max_return, 1),
            '捕获率%': round(combined_capture, 1),
            '评级': rating,
        })

    return pd.DataFrame(results)
```

## 风险与注意事项

1. **资金类指标数据依赖** — 主力连续净流出和融资余额衰减依赖 AKShare 的资金流数据，可能不稳定。第一版可先跳过这两个，用 12 个纯价格/量指标验证。

2. **卖出指标线显示复用** — `get_indicator_lines` 中卖出指标和买入指标共用同一套 TA-Lib 显示逻辑（如 MACD 顶背离和底背离都显示 DIF/DEA/HIST），只需在 `elif` 条件中追加卖出指标名。

3. **实际峰价计算** — `_compute_actual_peaks` 需要在 `run_sell_backtest` 和 `run_combined_backtest` 中都调用。确保 `df` 已按 `trade_date` 排序并设为索引。

4. **缓存隔离** — `_build_stock_ref_lows` 的缓存 key 需加 `'low'` 后缀，避免与 `_build_stock_ref_highs` 的缓存冲突。
